# Dive screenshots

Use the `screenshots` Maven profile in the `cli` module to run `hardwood dive`
headlessly via TamboUI recording and overwrite the fixed docs image files under
`docs/content/assets/cli/`.

Scripted UI navigation tape:
`docs/screenshots/dive-screenshots.tape`

The tape drives key presses through the `dive` screens and captures:
- `docs/content/assets/cli/01-landing-overview.svg`
- `docs/content/assets/cli/02-schema-tree.svg`
- `docs/content/assets/cli/03-1-rg.svg`
- `docs/content/assets/cli/03-2-rg-detail.svg`
- `docs/content/assets/cli/03-3-rg-column-chunks.svg`
- `docs/content/assets/cli/03-4-rg-column-chunk-detail.svg`
- `docs/content/assets/cli/04-pages-header-modal.svg`
- `docs/content/assets/cli/05-dict-search.svg`
- `docs/content/assets/cli/06-data-scrolled-right.svg`

Default fixture file:
`cli/src/test/resources/dive_screenshots_fixture.parquet`

The fixture is generated by `tools/simple-datagen.py` alongside the rest of the
checked-in test corpus. Regenerate it (run from the project's pinned virtualenv,
per [CONTRIBUTING](../../CONTRIBUTING.md) — the checked-in bytes depend on the
pinned PyArrow; this rewrites every generated fixture, not just this one):

```bash
source .docker-venv/bin/activate && python tools/simple-datagen.py
```

Run:

```bash
./mvnw -pl cli -am -Pscreenshots package -Dquick
```

Override fixture path:

```bash
./mvnw -pl cli -am -Pscreenshots package -Dquick -Dscreenshots.parquetFile=/absolute/path/to/file.parquet
```

Notes:
- Existing screenshots are overwritten in place.
- The fixture parquet file must already exist before running the profile.
- When `-Dscreenshots.parquetFile=...` is provided, that fixture is used directly.
